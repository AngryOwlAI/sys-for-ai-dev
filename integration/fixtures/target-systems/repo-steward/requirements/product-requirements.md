# Product Requirements

- `RS-REQ-001`: The target shall inspect only paths authorized by the current
  transaction permission envelope.
- `RS-REQ-002`: The target shall produce a proposal before any repository
  mutation.
- `RS-REQ-003`: Mutation, publication, merge, deletion, or authority change
  shall require a separate accountable approval and capable host.

These fixture requirements are candidate test inputs, not stakeholder-approved
requirements for a real repository service.
