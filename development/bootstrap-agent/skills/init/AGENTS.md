# Init Agent Guidance

Use this skill as the first routing layer for Sys4AI adoption.

Required discipline:

- Inspect available evidence before asking questions.
- Preserve `Sys4AI-dev`, `Sys4AI/`, `development/bootstrap-agent/skills`, `.codex/skills`, and `Sys4AI/assets/skills` boundaries.
- Treat brownfield first-pass work as read-only.
- Ask explicit approval before creating controlled artifacts or mutating a repository.
- Preserve candidate requirement labels as `REQ-CAND-*` or `NFR-CAND-*`.
- Preserve candidate strategic-intent labels as `VISION-CAND-*` or `VALUE-CAND-*`; record source or inference state, missing stakeholders, approval principal, conflicts, anti-values, waivers, and review cadence.
- Do not treat silence, model authorship, or structural validation as strategic-content approval.
- Route to existing downstream skills instead of duplicating their full behavior.
