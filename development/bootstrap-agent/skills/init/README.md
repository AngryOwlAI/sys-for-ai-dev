# Init

`/init` is the Sys4AI-dev runtime front door for system definition and adoption.

It classifies the situation, inspects available evidence, identifies candidate vision, value, stakeholder, approval, conflict, waiver, and review gaps for new or substantially changed targets, summarizes the route, and asks for approval before writing discovery records, Product Requirements Documents, implementation plans, or scaffolds.

Strategic content remains `VISION-CAND-*` or `VALUE-CAND-*` discovery evidence until an accountable human approves a separate registered artifact. Silence, model authorship, and structural validation are not approval.

The brownfield first pass is inspection-only. It may summarize repository evidence, but it must not modify the repository until the user or Director approves the next controlled artifact.
