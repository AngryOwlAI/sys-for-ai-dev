# Evaluation Harness Designer

Use this skill when behavior needs repeatable evaluation beyond a single manual review or unit test.

## Procedure

1. Define the behavior, risk, or quality target under evaluation.
2. Choose scenarios, counterexamples, and failure probes.
3. Specify measurable signals, rubric levels, and regression cadence.
4. Separate exploratory evaluation from acceptance gates.
5. Record limitations and the next evidence-producing command or artifact.

## Boundary

An evaluation harness design is not an evaluation result. Report only executed results as evidence.
