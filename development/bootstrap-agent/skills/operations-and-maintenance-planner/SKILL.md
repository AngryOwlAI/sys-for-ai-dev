# Operations And Maintenance Planner

Use this skill when a generated or maintained system needs runtime ownership, monitoring, incident, update, or evaluation planning.

## Procedure

1. Define the operational boundary and responsible owner.
2. Identify monitoring signals, failure modes, and incident paths.
3. Define update, regression, and evaluation cadence.
4. Record maintenance artifacts, retention expectations, and handoff points.
5. List unresolved readiness gaps.

## Boundary

Operational planning does not prove production readiness. Readiness depends on implemented controls and verified evidence.
