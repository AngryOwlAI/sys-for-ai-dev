# Context Window And Handoff Manager

Use this skill when long-running work needs a checkpoint, resumable handoff, temp handoff, or context-window risk assessment.

## Procedure

1. Identify the active project authorization, goal, latest handoff, and validation state.
2. Check whether context, time, or uncertainty requires a handoff.
3. Preserve source authority, open risks, changed files, validation commands, and next route.
4. Write the handoff in the controlled location authorized by explicit project authority.
5. Ensure the next agent can resume from tracked state without relying on chat memory.

## Boundary

A handoff is evidence and routing support. It does not complete work unless the required artifacts and validators prove completion.
