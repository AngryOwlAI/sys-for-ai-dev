# Threat Model And Permission Scope

Use this skill when a system, project authorization, integration, memory surface, or tool gains new permissions, autonomy, or data exposure.

## Procedure

1. Define the system boundary, actors, assets, data flows, and tools.
2. Identify permissions, trust assumptions, and misuse paths.
3. Map each material risk to an owner, control, and verification method.
4. Classify residual risk as accepted, mitigated, deferred, or blocked.
5. Record permission limits in the handoff or control artifact.

## Boundary

This skill scopes and records risk. It does not authorize unsafe permissions without a Director decision or equivalent owner acceptance.
