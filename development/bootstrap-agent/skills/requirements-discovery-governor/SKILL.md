# Requirements Discovery Governor

Use this skill when discovery records, stakeholder inputs, or early requirements need governance before becoming baseline requirements.

## Procedure

1. Identify the discovery source and intended downstream artifact.
2. Check stakeholder coverage, assumptions, scenarios, interfaces, risks, and open questions.
3. Decide whether the record is ready, needs more elicitation, or must be blocked.
4. Preserve discovery gaps instead of turning them into requirements.
5. Run the discovery validator or record why it cannot yet run.

## Boundary

Discovery governance protects the transition from user wants to requirements. It does not replace stakeholder elicitation.
