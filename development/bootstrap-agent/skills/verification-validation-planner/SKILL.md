# Verification Validation Planner

Use this skill when requirements, implementation plans, or acceptance packets need explicit verification and validation evidence.

## Procedure

1. Identify the requirement, decision, or acceptance criterion being verified.
2. Define the expected evidence type: test, validator, review, trace row, or external proof.
3. Separate planned checks from checks that have actually run.
4. Map each requirement to at least one concrete evidence path or an explicit gap.
5. Run or name the relevant validation gate and record residual uncertainty.

## Boundary

Verification planning does not prove completion by itself. Completion requires executed evidence or a clearly recorded gap.
