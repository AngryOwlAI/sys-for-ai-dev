# Interface And Integration Discovery

Use this skill when system boundaries depend on APIs, files, workflows, services, tools, people, or other external systems.

## Procedure

1. Identify each external dependency and interface surface.
2. Record data exchanged, direction, format, cadence, owner, and trust boundary.
3. Separate confirmed interfaces from assumed or proposed interfaces.
4. Identify integration risks, missing contracts, and validation needs.
5. Feed confirmed interfaces into architecture, threat, and artifact-contract work.

## Boundary

Interface discovery documents the boundary. It does not implement or authorize integrations.
