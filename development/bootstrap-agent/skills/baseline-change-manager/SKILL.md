# Baseline Change Manager

Use this skill when a controlled artifact, registry, requirement, generated derivative, or runtime surface changes baseline status.

## Procedure

1. Identify the current baseline and the proposed replacement or change.
2. Classify the change as additive, superseding, migration, rollback, or deprecation.
3. List affected canonical and generated surfaces separately.
4. Define validation, compatibility, and rollback evidence.
5. Record the new state in controlled registries or handoffs.

## Boundary

Do not rewrite activated control history. Supersede it with explicit evidence.
