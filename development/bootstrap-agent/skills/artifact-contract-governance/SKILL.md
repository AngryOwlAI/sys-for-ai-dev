# Artifact Contract Governance

Use this skill when an explicit project authority creates, changes, or audits contracts for controlled artifacts such as PRDs, handoffs, receipts, registries, schemas, or generated derivatives.

## Procedure

1. Identify the artifact type, producer role, consumer roles, and authority class.
2. Confirm required sections, validation contract, registry requirements, and derivative surfaces.
3. Check whether a proposed artifact is source authority or generated navigation.
4. Run artifact-contract validation.
5. Record contract changes and unresolved risks in the completion receipt.

## Boundary

Do not create a broad contract when a narrow existing contract can be reused. Generated derivative contracts must preserve their noncanonical status unless explicitly promoted.
