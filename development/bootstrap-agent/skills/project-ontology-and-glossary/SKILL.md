# Project Ontology And Glossary

Use this skill when project language needs precise definitions, term ownership, or ontology boundaries.

## Procedure

1. Identify the ambiguous or overloaded term.
2. Find its source usage and any existing definition.
3. Define the term, scope, exclusions, and related terms.
4. Separate core vocabulary from project-specific vocabulary.
5. Record source paths and unresolved terminology gaps.

## Boundary

Terminology control clarifies language. It must not silently introduce new requirements or domain ontology.
