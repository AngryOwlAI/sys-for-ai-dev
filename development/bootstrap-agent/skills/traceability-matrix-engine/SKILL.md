# Traceability Matrix Engine

Use this skill when an explicit project authority requires adding, repairing, or auditing trace rows linking user intent, requirements, implementation artifacts, validation evidence, and handoffs.

## Procedure

1. Identify the source requirement, decision, or intent being traced.
2. Map each claim to exact evidence paths rather than broad document references.
3. Mark coverage status, trace class, and semantic review verdict explicitly.
4. Check for duplicate or missing coverage.
5. Run trace validation and report residual gaps.

## Boundary

Traceability records evidence; it does not manufacture evidence. If evidence is missing, mark the gap directly.
