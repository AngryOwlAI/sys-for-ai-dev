# Director Decision Governor

Use this skill when tracked state requires a Director Decision Record to authorize a route, expand authority, record explicit project authority, supersede a decision, or close a decision.

## Procedure

1. Confirm why a Director decision is required.
2. Inspect the latest program state, handoff, implementation plan, and relevant registries.
3. Select one route and explicitly reject unsafe or out-of-order routes.
4. Bind the decision to one documented authorization or a documented stop.
5. Validate the decision record and registry row.

## Boundary

Do not use a Director decision to bypass human gates, act beyond explicit project authority, or mutate completed control records without supersession.
