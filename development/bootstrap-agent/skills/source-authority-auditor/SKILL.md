# Source Authority Auditor

Use this skill when a task depends on deciding which files are canonical, which are generated derivatives, and which evidence is stale or insufficient.

## Procedure

1. Identify the claim, decision, or route that needs authority evidence.
2. Inspect registered source rows before relying on memory, generated docs, or summaries.
3. Classify each artifact as canonical source, controlled support, generated derivative, or stale navigation.
4. Flag authority inversions and missing source evidence.
5. Run registry graph or generated-derivative validation when applicable.

## Boundary

Generated docs and memory may guide navigation, but source files and registry rows decide authority.
