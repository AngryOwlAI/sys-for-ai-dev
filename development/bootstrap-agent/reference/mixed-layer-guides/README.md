# Sys4AI Development Guides

This directory contains controlled explanatory guides for understanding how
Sys4AI lifecycle phases, roles, and skills relate to one another. The guides
also use the fictional HarborLight project to make those relationships
concrete without creating target-system authority.

## File guide

| File | Description |
|---|---|
| [`README.md`](README.md) | Provides this directory overview, file inventory, and suggested reading order. It is a navigation aid, not an independent authority source. |
| [`project_role_skills_overview.md`](project_role_skills_overview.md) | Provides the comprehensive, cross-cutting view of Sys4AI roles, skills, lifecycle stages, artifacts, authority boundaries, and execution relationships. It includes complete registered inventories, the HarborLight end-to-end example, current gaps, and practical selection checklists. |
| [`roles_guide.md`](roles_guide.md) | Focuses on all registered Sys4AI roles and the illustrative HarborLight project roles. It describes each role's responsibilities, collaborators, lifecycle relevance, skill use, handoffs, and safe interpretation boundaries. |
| [`skills_guide.md`](skills_guide.md) | Focuses on all registered Sys4AI product skills and their active Sys4AI-dev runtime counterparts. It explains dependencies, role bindings, lifecycle use, runtime availability, and illustrative HarborLight applications. |
| [`phases_guide.md`](phases_guide.md) | Focuses on the accepted eight-stage target-system lifecycle: Design, Develop, Implement, Test, Run, Maintain, Improve, and Retire. It explains phase contracts, transitions, participating roles and skills, artifacts, gates, and the HarborLight lifecycle example. |

## Suggested reading order

1. Start with [`project_role_skills_overview.md`](project_role_skills_overview.md)
   for the complete relationship model.
2. Use [`roles_guide.md`](roles_guide.md),
   [`skills_guide.md`](skills_guide.md), or
   [`phases_guide.md`](phases_guide.md) when you need a focused reference.

## Authority boundary

These guides explain registered Sys4AI authority; they do not replace the
governing PRDs, registries, execution bindings, skill manifests, or controlled
decisions linked from each guide. When a guide and a governing source differ,
use the governing source and treat the guide as requiring review.
