# Domain Pack Router

Use this skill when a task starts requiring domain-specific knowledge beyond the core Sys4AI framework.

## Procedure

1. Identify the domain pressure and why core skills are insufficient.
2. Check available domain-pack candidates or record that none exists.
3. Decide whether to route, defer, request domain input, or continue with core methods.
4. Keep project-specific constraints out of core skill files.
5. Record the routing decision and residual risk.

## Boundary

This skill routes to domain packs. It does not contain domain-pack content.
