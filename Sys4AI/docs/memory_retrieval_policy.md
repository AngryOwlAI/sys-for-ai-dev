# Memory Retrieval Policy

**Status:** Draft  
**Scope:** Source-first memory for `Sys4AI` Phase 1

---

## Policy

Memory retrieval is navigation, not authority.

A memory lookup may identify a useful source, registry row, control record, generated derivative, or local support artifact. It does not decide truth, scope, permission, or completion.

## Required practice

1. Run memory status before routing work.
2. Use lookup or deterministic search to locate candidate sources.
3. Inspect canonical sources or registry rows before using a hit for requirements, routing, AgentJob boundaries, handoffs, permissions, claims, or continuation state.
4. Record memory preflight evidence when memory affects routing.
5. Treat generated derivatives, semantic extracts, local vaults, and caches as noncanonical unless promoted through source authority workflow.

## Authority rule

Memory finds. Sources decide.
