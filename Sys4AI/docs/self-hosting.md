# Self-hosting

Self-hosting is a controlled release relation, not an identity collapse.

1. A bootstrap host builds and validates a trusted product release.
2. The trusted release is installed separately.
3. That release may produce a successor in a separate candidate workspace.
4. Independent verification evaluates the candidate against the same invariant
   suite and protected self-change probes.
5. A human release owner decides promotion while preserving rollback.

The candidate cannot approve its own purpose, authority, permission expansion,
evaluation threshold, safety waiver, production status, or release. The fixed
point is reached only when a trusted release can reproducibly produce a
candidate satisfying the invariant suite without hidden bootstrap behavior.
