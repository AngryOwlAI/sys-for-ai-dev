# Lifecycle phases guide

Sys4AI uses eight lifecycle stages: Design, Develop, Implement, Test, Run,
Maintain, Improve, and Retire. They are controlled states, not directory names
or automatic consequences of running a command.

- Design clarifies intent, stakeholders, boundaries, scenarios, and candidate
  requirements.
- Develop establishes accepted requirements, architecture, interfaces, plans,
  contracts, and evaluation obligations.
- Implement performs approved changes through bounded transactions.
- Test gathers verification and validation evidence for named claims.
- Run requires separate operational authority, monitoring, and incident paths.
- Maintain controls updates, dependencies, configuration, and reliability.
- Improve proposes evidence-grounded changes without bypassing authority.
- Retire withdraws operation, preserves required records, and handles data and
  dependency closure.

Transition evidence never substitutes for the accountable transition decision.
See `contracts/policies/lifecycle.md` and
`contracts/schemas/lifecycle.schema.json`.
